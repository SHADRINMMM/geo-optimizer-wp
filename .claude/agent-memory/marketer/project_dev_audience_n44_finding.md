Валидация весов скоринга (2026-07-02, ветка v2) на выборке n=44 доменов: on-site readiness score практически НЕ коррелирует с реальным цитированием в AI (Pearson r ≈ -0.08, Spearman ρ ≈ -0.03). 86% доменов в выборке — 0 цитат за окно измерения. Цитирование почти целиком объясняется brand prominence: high-prominence домены ~0.16 доли цитируемых prompts, все прочие группы ~0.

**Метод (важно для честности при упоминании):** Claude-only citation-прокси (один движок в этом раунде, не мультимотор), один временной срез (не лонгитюд/до-после), content_citability как категория тогда не измерялась отдельно от technical score. Источник: agent-memory `ai_score_v2_rebuild.md` (запись 2026-07-02 "Update A/B/C пост-деплой").

**Why:** это единственный на рынке честный сигнал против всех конкурентов (Profound, Otterly, Semrush AI Toolkit и т.д.), которые продают "оптимизируй on-page → получишь больше цитат" как гарантию. Наш собственный README CLI уже пишет: "score measures crawlability... does not guarantee citations — no tool can" — значит месседж этой находки не противоречит продукту, а прямо продолжает его честную рамку.

**How to apply:** это ГЛАВНЫЙ актив для dev-аудитории (HN/dev.to/IH/Reddit r/SEO, r/bigseo) — тон "инженер делится неудобными данными", не продавец. Всегда сопровождать честными ограничениями (n маленькое, один движок, один срез, prompt-selection not fully blind) — НЕ выдавать за строгое исследование. Что при этом ОСТАЁТСЯ сильным и почти механическим фактом (не корреляционным): robots.txt блокировка бинарна (заблокировал AI-бота → 0 цитат гарантированно), ~89% сайтов блокируют хотя бы одного AI-бота по ошибке. FAQ schema влияет на "как цитируют", не на "цитируют ли вообще".
