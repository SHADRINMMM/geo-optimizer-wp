Причина серии: Tilda-гайд (русский) по паттерну "честный гайд на локальном языке для региональной e-com/сайт-платформы" принёс 3 органические конверсии из Яндекса. Ресёрч 2026-07-07 (`/tmp/.../en-latam-research-2026-07-07.md`) искал аналоги в других регионах по критериям: (1) платформа регионально-эксклюзивна, не глобальный игрок, (2) есть штатный доступ к robots.txt и head-коду без доплаты, (3) нет конкурирующего platform-specific AI-visibility контента на местном языке.

Статус по кандидатам (на 2026-07-07):
- **Ticimax (Турция, tr)** — построен `nextjs-landind` `/guides/ticimax`, ветка `ticimax-guide-2026-07-07` (worktree `.claude/worktrees/ticimax-guide`), НЕ запушено. Доступ к robots.txt и head подтверждён официальной документацией (destekalani.com, ticimax.com/ticimax-seo) — WebFetch/WebSearch к этим доменам работали нормально (в отличие от Nuvemshop, см. [[blocker-nuvemshop-websearch-access]]).
- **Tiendanube/Nuvemshop (LATAM, es/pt-BR)** — построен `nextjs-landind` `/guides/nuvemshop` (контент pt-BR), ветка `nuvemshop-guide-2026-07-07` (worktree `.claude/worktrees/nuvemshop-guide`), НЕ запушено. WebFetch к доменам платформы всё ещё заблокирован (см. [[blocker-nuvemshop-websearch-access]]), но WebSearch (не WebFetch) дал достаточно сходящихся фактов из десятков независимых запросов, включая прямые цитаты из `atendimento.nuvemshop.com.br` — ручной ресёрч оператора не потребовался. Ключевой факт: доступ к коду темы (HTML/head) начинается с плана Impulso (3-й платный тариф, ~R$139-164/мес), Começo/Essencial — только косметический CSS. robots.txt не редактируется на ЛЮБОМ плане (не проблема — дефолт не блокирует AI-ботов). llms.txt в корень домена невозможен ни на одном плане (нет обхода, в отличие от Tilda).
- **Bumpa (Африка)** — отсеян: платформа технически не даёт доступа к коду/head/robots.txt вообще, гайд физически невозможен.

**Why:** без этой памяти легко забыть, что Nuvemshop и Ticimax — разные по риску (у Ticimax верификация через официальные источники прошла чисто, у Nuvemshop систематически блокируется WebFetch и требует другого подхода).

**How to apply:** если попросят повторить паттерн для новой платформы — сначала проверить эту память на предмет того, не отсеяна ли платформа уже, и учитывать при выборе метода верификации (WebFetch/WebSearch vs эскалация оператору за ручным доступом).
