При задаче 2026-07-07 (гайд по AI-видимости для Nuvemshop, pt-BR) WebFetch не смог получить контент ни с одного домена Nuvemshop/Tiendanube: atendimento.nuvemshop.com.br, ayuda.tiendanube.com, help.tiendanube.com, www.nuvemshop.com.br, www.tiendanube.com, dev.tiendanube.com, developers.tiendanube.com, tiendanube.github.io — включая простые запросы к корню домена без пути. WebSearch по тарифам/коду темы/robots.txt тоже не возвращал содержательных сниппетов с фактами (только общие/нерелевантные результаты).

**Why:** похоже на anti-bot/CDN-защиту на уровне всего домена (не конкретных страниц) — это НЕ временный сетевой сбой отдельного запроса, паттерн повторился на 40+ разных URL и запросов подряд.

**How to apply:** для будущих задач с Nuvemshop/Tiendanube как источником — сразу предупреждать оператора, что нужен ручной доступ (открыть страницу самому и вставить текст/скрин, или дать альтернативный источник — сторонний обзор/блог агентства, явно помеченный как вторичный). Не тратить десятки попыток WebFetch на эти домены повторно — сначала 2-3 контрольные попытки, при провале эскалировать сразу. См. также аналогичный урок из Tilda-гайда — нельзя писать гайд на неподтверждённых предположениях о тарифах/возможностях платформы (риск повторить ошибку "обещали невозможное").
